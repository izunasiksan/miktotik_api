6. Kenapa gap tidak terdeteksi

Karena tidak ada logic seperti ini:

expected_samples = duration / interval
actual_samples = count(*)

missing_samples = expected - actual
gap_ratio = missing / expected
7. Contoh solusi (SQL level)

Tambahkan count(*) di query:

func.count(BoardResourceStat.cpu_load).label("sample_count")

Contoh:

select(
    BoardResourceStat.board_id,
    func.avg(BoardResourceStat.cpu_load).label("avg_cpu"),
    func.max(BoardResourceStat.cpu_load).label("max_cpu"),
    func.count(BoardResourceStat.cpu_load).label("samples")
)

Lalu di Python:

expected = 1440
gap = expected - samples
gap_ratio = gap / expected

9. Problem lain yang cukup serius

Di bagian ini:

p95_stmt = text("""
SELECT PERCENTILE_CONT(0.95) ...
""")

Query ini dijalankan di dalam loop interface.

Kalau ada:

200 board
8 interface

maka query jadi:

1600 query tambahan

Ini bisa jadi N+1 performance issue.

Lebih efisien jika dihitung langsung di query utama.

10. Kesimpulan

Gap tidak teragregasi karena pipeline kamu hanya melakukan:

AVG
MAX
MIN
SUM

tanpa menghitung:

sample_count
expected_samples
missing_samples
gap_ratio

Akibatnya:

statistik bisa bias

kualitas data tidak diketahui

monitoring reliability tidak terlihat
